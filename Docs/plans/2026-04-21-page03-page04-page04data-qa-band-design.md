# Page03 Page04 Page04Data QA Band Design

## Goal

在 `page_03`、`page_04`、`page_04_data` 三页之间建立一条连续的底部问答带：

- `page_03` / `page_04`：显示同一组问题，右上角是 `?`
- `page_04_data`：显示阶段性解答，右上角是 `!`

其中 `Metal？` 不是当页结论，而是一个留给后文继续解释 `binary archive 2` 机制的钩子。

## Context

- `page_03` 与 `page_04` 当前主图底部仍有约 `192px` 可用空间，可以稳定放入一条底部信息带。
- `page_04_data` 当前表格实现为 `x=40, y=200, w=1200, h=456`，底部仅剩约 `64px`，无法直接在表格下方新增答案框。
- 当前仓库已经有成熟的 `StageBox` / `CalloutBadge` 原语，也已经有问题卡和问号 badge 的视觉语言，可以复用，不必重造一套组件风格。

## Approaches

### 方案 A：统一底部问答带，数据页上提表格

- 三页都使用同一尺寸、同一位置的底部卡片带，形成跨页连续语义。
- `page_03` / `page_04` 卡片内容相同，保留问题。
- `page_04_data` 保持答题页语义，但把表格整体上提并略收高度，为底部答案框腾出空间。

优点：

- 语义最连贯，观众能明确感受到“前两页提问，数据页先给一版解释”。
- 几何上容易控：page3/4 直接利用底部留白，page_04_data 只调整一个表格框。
- 与仓库 continuity 规则更一致，问题框和答案框可以复用同一承载带。

缺点：

- `page_04_data` 需要同步更新表格几何参数和对应文档。

### 方案 B：page3/4 底部问题框，page_04_data 右侧答案框

- 保留 `page_04_data` 现有表格几何。
- 把答案框缩成一个右侧小卡片，与表格并排。

优点：

- 不需要动数据页表格高度。

缺点：

- 与用户说的“下面加”不一致。
- 三页的问答带失去统一连续性。
- 数据页宽度本来就紧，右侧再塞答案卡容易牺牲表格可读性。

### 方案 C：page3/4 只加问号 badge，不加卡片正文；page_04_data 单独写答案框

- 最小改动，不新增问题卡正文。

优点：

- 实现最快。

缺点：

- 问题内容不会在第三、四页真正落地，不符合用户要求。

## Recommendation

推荐 **方案 A**。

它最符合用户意图，也最符合当前仓库强调的“树结构/连续语义/数学先行”的工作方式：

- `page_03` / `page_04` 在底部复用同一个问题卡
- `page_04_data` 在同一位置切换为答案卡
- `Metal？` 保持问句形态，作为后续解释 `binary archive 2` 的钩子

## Proposed Copy

### `page_03` / `page_04` 问题框

- 为什么需要预编译着色器？
- 为什么不能分发 binary？

### `page_04_data` 解答框

- GPU 编译依赖本地环境：OS / 驱动 / 芯片组合不同。
- binary 可能都不一样，而且编译很慢。
- Metal？后面再看 binary archive 2。

## Geometry Plan

### 连续问答带

- 统一问答带：`x=80, y=556, w=1120, h=120`
- 右上角 badge 锚点：`x=1172, y=580`
- 这会在 page3/4 底部主图下方保留约 `28px` 以上的安全间距。

### `page_04_data` 表格回收

- 现状：`x=40, y=200, w=1200, h=456`
- 调整后建议：`x=40, y=180, w=1200, h=336`
- 这样表格底部落在 `516`，与问答带顶部 `556` 之间保留 `40px` 缓冲。
- 5 行数据的行高约为 `(336 - 74) / 5 = 52.4px`，配合当前 `22px` 文本仍可读。
