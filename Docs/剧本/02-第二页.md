## 页面目标

把第一页的 `Input -> f(x) -> Output` 具体化成更接近图形渲染语义的版本，但仍然保留第一页的三段式空间骨架。

## 页面改动表

| 项目 | 内容 |
| --- | --- |
| 页面标题 | VertexData -> GPU -> Pixels |
| 页面副标题 | 从抽象函数到图形语义 |
| 当前页核心信息 | 输入不再是抽象 `Input`，而是 `VertexData`；中间不再是抽象 `f(x)`，而是 `GPU`；输出变成像素块 |
| 与上一页关系 | 不是重画，而是在相同三槽位里做内容演化 |
| 这一页新增了什么 | `VertexData` 的两三角形图形表达、单层 `GPU`、`4x4` 像素输出，以及 `GPU` 上方三个轻量 `U` 型参数接口伏笔 |
| 这一页删除了什么 | `Texture`、弯线、GPU 内框、GPU 四周装饰豆豆 |
| 这一页保留了什么 | 左中右三段结构、两根直线箭头、GPU 作为中间核心 |
| 需要强调的元素 | `GPU` 仍然是全页中心；上方 `U` 型接口只做“可配置”的轻量提示 |
| 暂不实现 / 暂不出现 | 更多输入资源、更多渲染阶段、显式状态对象名称 |
