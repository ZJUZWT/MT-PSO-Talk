# 第 02 页

## 页面目标

把第一页的 `Input -> f(x) -> Output` 具体化成更接近图形渲染语义的版本，但仍然保留第一页的三段式空间骨架。

## 页面改动表

| 项目 | 内容 |
| --- | --- |
| 页面标题 | VertexData -> GPU -> Pixels |
| 页面副标题 | 从抽象函数到图形语义 |
| 当前页核心信息 | 输入不再是抽象 `Input`，而是 `VertexData`；中间不再是抽象 `f(x)`，而是 `GPU`；输出变成像素块 |
| 与上一页关系 | 不是重画，而是在相同三槽位里做内容演化 |
| 这一页新增了什么 | `VertexData` 的两三角形图形表达、单层 `GPU`、`4x4` 像素输出 |
| 这一页删除了什么 | `Texture`、弯线、GPU 内框、GPU 四周装饰豆豆 |
| 这一页保留了什么 | 左中右三段结构、两根直线箭头、GPU 作为中间核心 |
| 需要强调的元素 | `GPU` 仍然是全页中心 |
| 暂不实现 / 暂不出现 | 更多输入资源、更多渲染阶段、状态对象 |

## 动画剧本

| Beat | 时间 | 画面变化 | 屏幕元素 | 讲述词 / 备注 |
| --- | --- | --- | --- | --- |
| 1 | 01 -> 02 过渡开始 | `Input` 保持原位置，开始长高并替换内容 | 左侧 node | 不移动骨架，只替换语义 |
| 2 | 过渡中段 | `f(x)` 在原位置形变成 `GPU` | 中间 node | 让观众意识到抽象函数正在被具体化 |
| 3 | 过渡后段 | `Output` 在原位置替换成 `4x4` 像素块 | 右侧 node | 输出不再抽象，而是可见的像素结果 |
| 4 | 过渡结束 | 停在 `VertexData -> GPU -> Pixels` 的静态终态 | 全页 | 继续为后续 OpenGL / Vulkan 页面做准备 |

## Remotion 实现备注

| 项目 | 内容 |
| --- | --- |
| 需要新增的节点 / 元素 | 左侧两三角形图标、右侧 `4x4` 像素网格 |
| 需要复用的元素 | 第一页三槽位骨架、两根直线箭头、GPU 中轴位置 |
| 是否需要 draw.io 草稿 | 暂不需要 |
| 是否需要镜头推进 / 缩放 | 不需要 |

| 风险点 | 如果三槽位位置发生跳变，会破坏和第一页的连续性 |
