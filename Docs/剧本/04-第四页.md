# 第 04 页

## 页面目标

在保留第三页底部主轴和上方输入分区的前提下，把第 04 页改成真正的 Vulkan 语义：上方输入先进入 `Description`，再 create 出 `PSO`，真正高亮的 `GFXAPI CALL` 只剩下 `PSO -> GPU`。

## 页面改动表

| 项目 | 内容 |
| --- | --- |
| 页面标题 | Vulkan PSO |
| 页面副标题 | 先写进 Description，再创建出 PSO |
| 当前页核心信息 | `SPIR-V`、`Depth`、`Blend` 不再直接高亮地敲到 `GPU`，而是先进入 `Description`，再由橙色的 create API 生成 `PSO`，最后还剩 `PSO -> GPU` 这一条高亮运行时调用 |
| 与上一页关系 | 继承第三页的上下两层，只在中间插入 `Description / PSO` 这条创建语义，不重做整体骨架 |
| 这一页新增了什么 | 中层 `Description`、`PSO`，以及 `Description -> PSO` 的 create 关系 |
| 这一页删除了什么 | 第三页中 `ShaderBinary`、`Depth`、`Blend` 各自直达 `GPU` 的三条高亮调用 |
| 这一页保留了什么 | 底部 `VertexData -> GPU -> Pixels` 主轴、上方 `ShaderCode / Depth / Blend` 分区、左侧的 shader 产物链 |
| 需要强调的元素 | 橙色的 `Description -> PSO` create 调用，以及 `PSO -> GPU` bind 调用 |
| 暂不实现 / 暂不出现 | 更细的 `Raster` 状态、`RenderPass`、`DescriptorSet` 等后续 Vulkan 结构 |

## 动画剧本

| Beat | 时间 | 画面变化 | 屏幕元素 | 讲述词 / 备注 |
| --- | --- | --- | --- | --- |
| 1 | 03 -> 04 过渡开始 | 第三页原本 4 条橙色 API 调用开始缩短并退场 | 上方橙色线与编号 | 先让观众感受到“直接调用很多”的旧状态正在被收回 |
| 2 | 过渡中段 | 左侧 `ShaderBinary` 语义切换为 `SPIR-V`，三条向下线改成灰线 | 上方链路 | 先把 Vulkan 的上游输入讲准确，不再沿用 OpenGL 的词 |
| 3 | 过渡中段 | 中间先长出一个 `Description` | `Description` 节点 | 这页不是直接长出 PSO，而是先出现一份要被填写的创建描述 |
| 4 | 过渡后段 | `Description -> PSO` 变成橙色 create 调用 | `Description -> PSO` | 这一步明确告诉观众：create 本身也是一条 Graphics API |
| 5 | 过渡后段 | `PSO -> GPU` 作为第二条橙色调用保留下来 | 中层到 GPU | 强调 Vulkan 把运行时真正反复发生的绑定收敛成更少的一步 |
| 6 | 过渡结束 | 停在 `Vulkan PSO` 的静态终态 | 全页 | 让观众把“create 阶段显式”与“runtime bind 精简”同时记住 |

## Remotion 实现备注

| 项目 | 内容 |
| --- | --- |
| 需要新增的节点 / 元素 | `Description` 节点、`PSO` 节点、`Description -> PSO` 与 `PSO -> GPU` 两个橙色 badge |
| 需要复用的元素 | 第三页底部主轴、上方四个节点的位置、左侧 shader 产物链 |
| 是否需要 draw.io 草稿 | 当前可直接在网页里验证，因为重点是页间演化，而不是新拓扑探索 |
| 是否需要镜头推进 / 缩放 | 不需要 |
| 是否需要逐步显隐 | 需要，先退掉旧橙线，再显出 `Description`，随后显出橙色 create 线与最终 bind 线 |
| 风险点 | 如果仍然保留 `ShaderBinary` 这个词，或直接从上方跳到 `PSO` 而没有 `Description`，这一页就会继续像“OpenGL 换皮版 PSO” |
