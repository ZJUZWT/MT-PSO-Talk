# 第 04 页

## 页面目标

在保留第三页底部主轴和上方输入分区的前提下，在中间插入一个 `PSO`，把 Vulkan 的关键变化讲清楚：上方配置关系还在，但真正高亮的 `GFXAPI CALL` 已经明显减少。

## 页面改动表

| 项目 | 内容 |
| --- | --- |
| 页面标题 | Vulkan PSO |
| 页面副标题 | 从多条直接调用变成一个打包对象 |
| 当前页核心信息 | `ShaderBinary`、`Depth`、`Blend` 不再各自高亮地直达 `GPU`，而是先汇入 `PSO`，再由 `PSO -> GPU` 作为唯一高亮 API 调用 |
| 与上一页关系 | 继承第三页的上下两层，只在中间插入一个新的收口层，不重做整体骨架 |
| 这一页新增了什么 | 中层 `PSO` 节点、唯一一条高亮的 `PSO -> GPU` 调用 |
| 这一页删除了什么 | 第三页中 `ShaderBinary`、`Depth`、`Blend` 各自直达 `GPU` 的三条高亮调用 |
| 这一页保留了什么 | 底部 `VertexData -> GPU -> Pixels` 主轴、上方 `ShaderCode / ShaderBinary / Depth / Blend` 分区、`ShaderCode -> ShaderBinary` 左侧水平链 |
| 需要强调的元素 | `PSO` 本体，以及唯一保留的高亮 `PSO -> GPU` 调用 |
| 暂不实现 / 暂不出现 | 更细的 `Raster` 状态、`RenderPass`、`DescriptorSet` 等后续 Vulkan 结构 |

## 动画剧本

| Beat | 时间 | 画面变化 | 屏幕元素 | 讲述词 / 备注 |
| --- | --- | --- | --- | --- |
| 1 | 03 -> 04 过渡开始 | 第三页原本 4 条橙色 API 调用开始退场 | 上方橙色线与编号 | 先让观众感受到“直接调用很多”的旧状态正在被收回 |
| 2 | 过渡中段 | 中间长出一个 `PSO` | `PSO` 节点 | 不是删掉上方结构，而是在它们和 `GPU` 之间插入一个打包层 |
| 3 | 过渡中段 | `ShaderCode -> ShaderBinary` 和三条向下连接改成灰线 | 上方四条线 | 这些关系还在，但在这一页不再被讲成高亮的 `GFXAPI CALL` |
| 4 | 过渡后段 | 只剩下一条橙色 `PSO -> GPU` 竖线 | 中层到 GPU | 强调 Vulkan 风格把运行时显式调用压缩成更少的一步 |
| 5 | 过渡结束 | 停在 `Vulkan PSO` 的静态终态 | 全页 | 让观众把“多条直接调用”与“单一收口对象”建立对比 |

## Remotion 实现备注

| 项目 | 内容 |
| --- | --- |
| 需要新增的节点 / 元素 | `PSO` 节点、唯一的 `PSO -> GPU` 橙色 badge |
| 需要复用的元素 | 第三页底部主轴、上方四个节点的位置、左侧 `ShaderCode -> ShaderBinary` 链路 |
| 是否需要 draw.io 草稿 | 当前可直接在网页里验证，因为重点是页间演化，而不是新拓扑探索 |
| 是否需要镜头推进 / 缩放 | 不需要 |
| 是否需要逐步显隐 | 需要，先退掉旧橙线，再显出 `PSO` 与唯一保留的高亮调用 |
| 风险点 | 如果 `PSO` 不处在真正的中层，或 GPU 仍然和 `PSO` 同时抢焦点，这一页就会讲不出“调用减少”这个主题 |
