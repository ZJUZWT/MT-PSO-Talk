## Spatial constraints

- `Computer` 位于页面左下 1/3，`Phone` 位于页面右下 1/3，二者的视觉中心线必须落在同一水平线上，并且这条线要贴合页面水平中线。
- `.ushaderbytecode` 位于两台设备之间的桥位，不能只像一条居中的标题条。
- `Phone` 内部的 `vertex / GPU / pixels` 必须共享一条清晰的垂直中轴，构成稳定的内部流程骨架。
- 这一页上半区可以留白，但留白必须服务于“左右设备 + 中间载体”的三角关系，不能把中间节点做成孤岛。
- settled frame 不出现任何 page12 之后的新增资产或回流线。

## Element styles

- 设备外壳使用中性浅底与低饱和描边，不在这一页显示 `Computer / Phone` 文字标签。
- `.ushaderbytecode` 使用更强的强调描边，承担答案卡片角色。
- 只要存在连续语义，优先使用同一个 render carrier 做位置、尺寸、标签变化；禁止用“新卡片淡入”伪装 continuity。
- `Phone` 内部 runtime trio 延续 page10 的图形语言，不引入新图标家族。
