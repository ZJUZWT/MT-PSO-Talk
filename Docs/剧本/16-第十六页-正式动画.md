## 插页转场规则

1. 主流程页整体缩小并淡出，短暂保留为背景 memory plate。
2. onepage 内容板从中心轻微放大浮入，标题、内容卡、底部 takeaway 分两拍落位。
3. 离场时 onepage 整体缩小并淡出，短暂恢复上一张主流程静态帧，再进入下一张主流程动画。

## 版式与内容约束

- 四张文件卡必须像“字典页”，不是缩小版流程图。
- 每张卡的说明只允许覆盖“文件里装什么”和“和哪一步相关”，不要展开成实现细节段落。
- `OpenGL` 备注只在确实需要的两张卡上出现，不要平均撒在所有卡里。
- 底部 takeaway 必须把 `expand` 与 `build` 的分工压缩成两条短句。

## Acceptance checklist

1. 一眼读得出这是四类文件的说明页，而不是 stable 闭环正式出场页。
2. `rec.upipelinecache`、`scl.csv`、`stablepc.csv`、`stable.upipelinecache` 的职责不能互相串位。
3. `expand` 与 `build` 的差异必须能被单独复述。
4. 页面密度要低于主流程页，不能重新长出多段 routing。
