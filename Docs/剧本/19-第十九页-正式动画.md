## 元素样式约束

- `stable.upipelinecache` 用强调型文件节点，保持本页唯一强输入锚点。
- `UE PSO`、`内存中 PSO`、`硬盘中的 PSO` 用同一套浅底分组框，保证分组语义一致。
- 组标题、行标题和 `GPU` 字号要比上一版更大，保证投影阅读时不会显得发虚。
- 三个 API 内存节点与三个硬盘节点全部横向对齐，箭头端点必须居中。
- `OpenGL` 与 `Program Binary` 合并为同一节点的上下双行文案，不拆成两个块。
- `Metal` 节点必须同时出现 `Binary Archive` 与 `系统管理`；硬盘节点必须同时出现 `BinaryArchive` 与 `functions.data`。
- 主轴箭头使用小号三角头，缓存边使用虚线，不允许出现折线、偏心锚点或 oversized arrowhead。
